Tools

Git

Version control system to track code changes.
Official Documentation
▼

Git is a distributed version control system used in software development. It helps track changes, manage versions, and collaborate effectively. It supports branching, merging, and rollback features. Developers can work independently and merge code later. It maintains a complete history of changes and improves teamwork. Git is widely used in modern DevOps pipelines.

Where to use:
  • Source code management
  • Team collaboration
  • Version tracking
  • Software development projects

GitHub

Platform to host and manage Git repositories.
Official Documentation
▼

GitHub is a cloud-based platform for hosting Git repositories. It enables developers to collaborate through pull requests, issues, and workflows. It provides version control, CI/CD integration, and project management features. GitHub is widely used for open-source and private projects.

Where to use:
  • Code hosting
  • Team collaboration
  • Project management
  • CI/CD integration

Jenkins

Automation server for CI/CD pipelines.
Official Documentation
▼

Jenkins is an open-source automation tool used to build, test, and deploy applications. It supports plugins for integration with many tools. Jenkins pipelines automate the software lifecycle and reduce manual work. It helps detect bugs early and improves delivery speed.

Where to use:
  • CI/CD pipelines
  • Automated testing
  • Deployment automation
  • Build management

Docker

Container platform for consistent environments.
Official Documentation
▼

Docker is a containerization platform that packages applications with dependencies. It ensures applications run consistently across environments. Docker containers are lightweight, portable, and scalable. It simplifies deployment and reduces compatibility issues.

Where to use:
  • Application deployment
  • Microservices architecture
  • Development environments
  • Cloud applications

Kubernetes

Container orchestration and scaling tool.
Official Documentation
▼

Kubernetes is used to manage containerized applications. It automates deployment, scaling, and load balancing. It ensures high availability and reliability. Kubernetes handles failures and manages clusters efficiently in production environments.

Where to use:
  • Cloud deployments
  • Microservices management
  • Auto-scaling systems
  • High-availability applications

Ansible

Automation tool for configuration management.

▼

Ansible automates IT tasks such as configuration, deployment, and provisioning. It uses simple YAML playbooks and requires no agent installation. It is efficient, scalable, and widely used in DevOps environments.

  • Application deployment
  • Automation tasks
  • Infrastructure management
